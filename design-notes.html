Design Notes

Sections

Optional featured cards, latest cards added, pinned cards which are for the author for his own personal favorite cards.

Cards

  1. Top row of all cards should have card id on the left, and icons in the middle, and a drop down menu on the right.
  2. Heart icon for likes. Red like is like by the author/owner of the card, pink like is by other people but not the author. Below the icon is the total number of likes.
  3. Eye icon for number of views.
  4. If this card is not owned by the viewer, then a bell icon (or telescope) for getting notified of updates to this card.

Cards can be private, public, or view permissions granted to specific users. Should we ever allow other people to write to someone else's card? Seems hard to deal with. They could just create a shared account.

Login

Support for users logging into their own accounts using oath/google identification. Once user is logged in, then sending emails for updates will be easier to handle/authenticate/manage.

Each user can optionally create a profile page for him/herself. Profile page can also show stats about the account, e.g. how many cards the user has created, how many added last week, total views, total likes, when the user joined synapcards, etc.
